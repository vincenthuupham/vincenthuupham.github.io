This project was my introduction to microcontrollers and embedded systems. I had used an IDE before in my Java classes with IntelliJ IDEA, but this was my first time actually writing and flashing code directly through the Arduino IDE.

I used an ESP32 microcontroller to blink an LED by toggling a GPIO pin between HIGH (3.3V) and LOW (0V) at 1 second intervals.

## What I Learned
Before this project I had no experience with microcontrollers or embedded systems.

One thing that stood out was understanding what "embedded" actually means. Unlike code running on a laptop or phone, the ESP32 has no operating system. The Arduino IDE compiles C++ code into raw binary on your computer, flashes it onto the chip over USB, and the chip executes those instructions directly the moment it receives power, nothing in between.

This also explains why the ESP32 keeps blinking even after being unplugged and reconnected to a different power source. The code is written into the chip's flash memory, which is non-volatile and persists without power, so the moment the board receives electricity it boots up and runs the code automatically, no computer needed.

I also learned how a microcontroller interacts with physical components. The ESP32's GPIO pins can be configured in code as either inputs or outputs, allowing the board to either read sensors or control devices like LEDs and motors.

On the hardware side, the LED requires a current-limiting resistor to prevent damage, and the circuit must form a complete path between the output pin and ground for current to flow.

## Code
Arduino sketches are written in C++ and follow a two-function structure: `setup()` runs once on startup, and `loop()` runs continuously for as long as the board has power.

```cpp
#define LED 2        // assign pin 2 the name LED
void setup() {
  pinMode(LED, OUTPUT);  // configure pin 2 as an output
}
void loop() {
  digitalWrite(LED, HIGH);  // set pin 2 to 3.3V, LED on
  delay(1000);               // wait 1 second
  digitalWrite(LED, LOW);   // set pin 2 to 0V, LED off
  delay(1000);               // wait 1 second
}
```

## Code Walkthrough
- `#define LED 2` — assigns the name LED to pin 2 for readability
- `setup()` runs once on startup — configures pin 2 as an output so it can supply voltage
- `loop()` runs continuously — toggles the pin HIGH then LOW with 1 second delays, repeating indefinitely

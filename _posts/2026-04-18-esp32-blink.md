This project was my introduction to microcontrollers and embedded systems.

I used an ESP32 microcontroller to blink an LED by toggling a GPIO pin, one of the board's programmable electrical pins, between HIGH (3.3V) and LOW (0V) at 1 second intervals using the Arduino IDE.

Through this project I learned how a microcontroller interacts with physical components. The ESP32's GPIO pins can be configured in code as either inputs or outputs, allowing the board to either read sensors or control devices like LEDs and motors.

I also learned the importance of basic circuit components. The LED requires a current-limiting resistor to prevent damage, and the circuit must form a complete path between the output pin and ground for current to flow.

On the software side, the Arduino IDE compiles C++ code into machine code, which is then flashed onto the ESP32 over USB. This means the chip stores raw binary instructions and executes them directly with no operating system underneath. This is what makes it an embedded program. The code structure follows Arduino's standard format: a `setup()` function that runs once on startup, and a `loop()` function that runs continuously.

## Code

```cpp
#define LED 2        // assign pin 2 the name LED
void setup() {
  pinMode(LED, OUTPUT);  // configure pin 2 as an output
}
void loop() {
  digitalWrite(LED, HIGH);  // set pin 2 to 3.3V, LED on
  delay(1000);               // wait 1 second
  digitalWrite(LED, LOW);   // set pin 2 to 0V, LED off
  delay(1000);               // wait 1 second
}
```

## Code Walkthrough

- `#define LED 2` — assigns the name LED to pin 2 for readability
- `setup()` runs once on startup — configures pin 2 as an output so it can supply voltage
- `loop()` runs continuously — sets the pin HIGH (3.3V) to turn the LED on, waits one second, sets it LOW (0V) to turn it off, waits one second, then repeats indefinitely

## What I Learned

Before this project I had no experience with microcontrollers or embedded systems. The biggest thing I learned was understanding what "embedded" actually means. Unlike code running on a laptop or phone, there is no operating system on the ESP32. The Arduino IDE compiles the C++ code into raw binary on your computer, flashes it onto the chip over USB, and the chip executes those instructions directly the moment it receives power. The code is flashed onto the chip and runs directly on the hardware, with no operating system in between.
